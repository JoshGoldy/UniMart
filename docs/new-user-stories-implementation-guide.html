UniMart New User Stories Implementation Guide

Step-by-step rebuild guide for bringing the clean repo up to the current app state for the six newer user stories, now split into 7 commits.

How to use this document: this guide is written for your clean repo that stopped before the newer listing and password-recovery features. Each milestone points to an exact snapshot file stored in docs/commit-snapshots/new-user-stories/. Copy the snapshot into your clean repo, stage it, and commit it.

Important: these steps assume you are working inside the clean repo folder, not the current working UniMart app folder.

Snapshot Files

Snapshot Purpose
01-auth.js Auth/data layer with seller listing CRUD, but without image upload and without forgot password yet.
02-listings.html Live listing management page with add and edit support, but without delete/status management and without image upload yet.
03-listings.html Listing management page with delete and sold/active status management added, but still without image upload.
04-auth.js Auth/data layer with image upload support added, but still without forgot password yet.
04-listings.html Final listing management page with image upload and previews.
05-search.html Marketplace page with clickable cards, listing images, and a full detail modal.
06-auth.js Final auth/data layer including forgot password helpers.
06-login.html Final login page including forgot password request and password recovery flow.
