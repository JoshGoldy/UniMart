UniMart Clean Commit Plan

Prepared for rebuilding a clean repository history in a truthful, team-aligned way.

How to use this document: each commit below is a clean milestone for the new repo. The person listed under Primary owner should only make that commit if they actually handled or are reviewing that portion of the work in the new repo. The commands below are normal commit commands only; they do not change author identity.

This plan is based on the current UniMart app structure: authentication, profile management, shared shell UI, dashboard analytics, marketplace search, and UI polish.

Practical Notes For The New Repo

  • Create the new repository first, then rebuild the project in the milestone order above.
  • Only commit files that genuinely belong to that milestone to keep history clean.
  • If two people really worked together on a milestone, add a real Co-authored-by: trailer in the commit body.
  • Keep database SQL and RLS policy files under docs/ or a supabase/ folder so backend work is visible in the repo.
  • Have Nkosi maintain a short changelog or sprint notes file after each milestone to strengthen the project narrative.
