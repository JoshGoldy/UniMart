UniMart User Acceptance Tests

Implemented acceptance tests only, written in who-what-why format.

Document scope: this UAT document includes only the implemented acceptance tests for the current UniMart application. Any planned but incomplete requirements such as permanent account deletion, condition filtering, price range filtering, and purchase history are intentionally excluded.

User Story 1

As a new user, I want to create an account and log in using a username and password, so that I can securely access and manage my personal listings and activity.

UAT ID Who What Why Test Scenario Expected Result
UAT-1.1 As a new user I want to create an account with my details So that I can register for UniMart Enter valid sign-up details and submit the registration form User moves to OTP/email verification step
UAT-1.2 As a new user I want to verify my account with the OTP code So that my account becomes active and secure Enter the correct 8-digit OTP code Account is verified and user can continue
UAT-1.3 As a registered user I want to log in with my email and password So that I can access my account Enter valid credentials on the login page User is authenticated and redirected to marketplace
UAT-1.4 As a registered user I want the system to reject incorrect credentials So that my account remains secure Enter invalid login details Error message is shown and access is denied
UAT-1.5 As a new or registered user I want required fields to be validated So that I know what information is missing or incorrect Submit login or signup form with empty required fields Validation messages are displayed
UAT-1.6 As an unauthenticated user I want protected pages to block access until I log in So that private account pages remain secure Try opening dashboard, profile, or search pages without a session User is redirected to login page

User Story 2

As a registered user, I need to be able to view and change my username, email and password in a “My Profile” page.

UAT ID Who What Why Test Scenario Expected Result
UAT-2.1 As a registered user I want to open the My Profile page from the user menu So that I can manage my account details Click the top-right dropdown and select My Profile Profile page opens successfully
UAT-2.2 As a registered user I want to view my stored profile details So that I can confirm my account information Open My Profile page Current name, email, account type, and campus info are displayed
UAT-2.3 As a registered user I want to update my username and email So that my account information stays correct Edit full name and email, then save Changes are stored and reflected on the page
UAT-2.4 As a registered user I want to change my password So that I can keep my account secure Enter current password, new password, and confirmation, then save Password is updated successfully
UAT-2.5 As a registered user I want incorrect current passwords to be rejected So that nobody can change my password without proper verification Enter wrong current password and attempt update Error is shown and password remains unchanged
UAT-2.6 As a registered user I want to update my university, campus, and student number So that my profile stays relevant to my campus identity Edit campus info and save Campus details update successfully
UAT-2.7 As a registered user I want to sign out from my account So that I can securely end my session Click sign out from dropdown or profile page Session ends and user is redirected to login

User Story 3

As a user, I want to search and filter listings by category, so that I can quickly find items that match my preferences.

UAT ID Who What Why Test Scenario Expected Result
UAT-3.1 As a user I want to view available marketplace listings So that I can browse items offered on the platform Open the search page with listings in the database Active listings are displayed as cards
UAT-3.2 As a user I want to search listings by keyword So that I can quickly find relevant items Type a keyword into the search bar Matching listings are displayed
UAT-3.3 As a user I want to filter listings by category So that I can narrow results to the type of item I want Click a category chip such as Electronics or Furniture Only listings in the selected category are shown
UAT-3.4 As a user I want search and category filters to work together So that I can narrow listings more precisely Enter a keyword and select a category Only listings matching both filters are shown
UAT-3.5 As a user I want to be informed when no results match my search So that I understand there are no matching listings Use a search/filter combination with no results A no-results empty state is shown

User Story 4

As a user, I want to view a dashboard with analytics such as listing counts and summary charts, so that I can track my activity and performance on the platform.

UAT ID Who What Why Test Scenario Expected Result
UAT-4.1 As a user I want to open my dashboard So that I can see a summary of my activity Open the dashboard page Dashboard loads successfully
UAT-4.2 As a user I want to view my listing counts and totals So that I can track how many items I have active or sold Open dashboard with listings linked to my account Active Listings, Sold Listings, Active Listing Value, and Listed This Month are displayed
UAT-4.3 As a user I want to view summary charts So that I can understand my listing trends and category activity Open dashboard with listing data available Trend and category charts are displayed
UAT-4.4 As a user I want to view my recent listing activity So that I can quickly monitor my latest items Open dashboard with recent listings available Recent Listings panel is shown
UAT-4.5 As a user I want a meaningful empty state when I have no data So that I still understand the dashboard even without listings Open dashboard with no listings linked to my account Empty analytics state is displayed
Note

This document includes only implemented acceptance tests. Features that are not yet complete were removed to keep the UAT scope accurate and defensible.

Examples excluded from this version: permanent account deletion, condition filtering, price range filtering, and purchase history.
